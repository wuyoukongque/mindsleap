# MindsLeap Website Site Tracker

本文件用于管理官网改版、页面优化、交互调整、功能修复与技术改进。

`PROJECT_CONTEXT.md` 负责长期稳定规则；
`CONTENT_TRACKER.md` 负责新闻 / 洞察发布；
`SITE_TRACKER.md` 负责站点本身的迭代进程。

## Workflow

站点更新统一按以下状态推进：

1. `Backlog`
   已识别的问题或机会，尚未排期
2. `Planned`
   已明确目标、范围和优先级，准备执行
3. `In Progress`
   正在修改设计、代码或交互逻辑
4. `Review`
   已完成实现，待预览、验证或确认
5. `Done`
   已验证通过，并进入当前稳定基线

## Scope

这里适合记录的事项包括：

- 首页结构、文案与视觉改版
- 新闻页、详情页、列表页的交互优化
- SEO、metadata、社交分享图等站点级优化
- 组件重构、样式统一、响应式体验优化
- Bug 修复、性能问题、可维护性提升

不适合记录的事项：

- 单篇文章写作、改稿、头图制作、发布状态
- 临时素材收集
- 仅与单篇内容有关的编辑意见

## Stable Baseline

以下事项已完成或已作为当前基线确认，后续修改时不要误改回去：

- 首页新闻区按顺序展示两个模块：最新活动、最新洞察
- `/news` 页面默认 tab 为 `events`
- 点击新闻列表进入详情页时，滚动位置已调整，标题应保持可见

## Working Rules

- 站点改版任务尽量与内容发布任务分开提交
- 如果一次改动同时涉及设计、交互和文案，先明确主目标，再开始编码
- 影响首页、新闻页、详情页的改动，完成后应进行本地预览
- 修改站点行为前，先检查是否与 `PROJECT_CONTEXT.md` 中已确认规则冲突

**作者：王林Lincoln | MindsLeap创始人 | Founders Space合伙人 | 企业家AI俱乐部创始人**

「对我来说，编程这件事已经被解决了。」

说这句话的人，叫 Boris Cherny。

2026 年 5 月初，同样是在红杉资本（Sequoia Capital）主办的 AI Ascent 峰会上，这位 Claude Code 的创造者、Anthropic 工程师，坐在台上接受采访，语气平静地说出了一句在场所有工程师都听得耳朵发烫的话。

他在 2026 年没有手写过一行代码。

但他每天发出几十个 PR，最多的一天发了 150 个。

而且是从手机上完成的。

这不是一个技术极客的炫耀，而是一个正在发生的信号，关于软件行业的下一次结构性转变。

## 一个「意外」发明的产品

在讲他的工作方式之前，Boris 先讲了 Claude Code 诞生的故事。

2024 年底，他加入了 Anthropic 内部的一个小型孵化团队。那个团队只有几个人，使命是构建他们自己想要的东西——最后造出了三件事：Claude Code、MCP 协议，以及 Claude 桌面客户端。

「我们当时的判断是，」Boris 说，「模型的能力已经超出了现有产品能够捕捉到的边界。」

他用了一个词：**product overhang**（产品积压）。

意思是：模型能做的事，远超当时所有产品让它做的事。

当时最先进的编程辅助工具，是「自动补全」——你在 IDE 里按 Tab，模型帮你补全一行。Sonnet 3.5 让这件事变得可用。但 Boris 的判断是：下一步不是更聪明的补全，而是让 AI 直接**写完整段代码**，完整地把任务做完。

他造了出来。

然后在最初的六个月里，这个东西几乎没人用。

「它太难用了，不够好。我自己大概只用它来写 10% 的代码。」

直到 Opus 4 发布，指数级增长才开始。之后每一次模型迭代，Claude Code 都再次爆发一次。

这里有一个对企业家来说很重要的洞察：**Boris 是在为还不存在的模型构建产品。** 他不是在解决当下的问题，而是在押注「当模型足够好，这个产品会是人们最想要的东西」。

## 「编程已被解决」——他的意思是什么

Boris 在台上问了全场一个问题：谁现在 100% 手写代码？谁 100% 用 AI 写代码？

他属于后者。

他解释「编程已被解决」这句话时加了一个重要的限定：对**他写的那类代码**来说是这样。Claude Code 的代码库用 TypeScript 和 React 写成，没有什么奇特的技术栈，模型对这类代码非常熟悉，可以无障碍接手。

但他也承认，有些代码库还没到这一步——代码量太大、太复杂、用的语言太冷门。

「通常的答案是，等下一个模型。」

这句话值得反复咀嚼。不是「等一个更好的工具」，而是「**等下一个模型**」——意味着他们看问题的时间尺度是模型迭代周期，而不是季度路线图。

## 从手机上管理几百个 AI 并发工作

然后他展示了自己现在的工作方式。

打开手机上的 Claude app，左侧有一个 Code 标签页，里面是 5 到 10 个进行中的会话。每个会话里，同时跑着若干 Agent。

「我现在大概有几百个 Agent 在运行。每天晚上，有几千个在做更深层的任务。」

最让他着迷的，是一个叫 **`/loop`** 的功能。

逻辑极其简单：让 Claude 用 cron 设一个定时任务，每隔一段时间（每分钟、每五分钟、每天）自动重复执行一次。

Boris 现在有几十个这样的循环在运行：

- 一个在帮他盯着所有 PR，自动修 CI、自动 rebase
- 一个在保持代码库的测试健康
- 一个每 30 分钟从 Twitter 抓取用户反馈、聚类整理、推送给他

「我觉得 Loop 就是未来。如果你还没试过，强烈推荐。」

对我来说，这个工作方式让我想起了一件事：过去雇一个团队做的事情，现在变成了管理一组任务循环。人的角色，从执行者变成了**调度者**和**验收者**。

## 印刷机时刻：编程将变成人人都会的事

这是整场演讲里我最喜欢的一段。

有人在台下问：编程会不会像 Microsoft Office 一样，变成每个人都会的基础技能？

Boris 说：会的，而且会比那走得更远。

他用了一个历史类比：**1400 年代的印刷机。**

印刷机发明前，欧洲只有大约 10% 的人口识字。他们受雇于不识字的国王和贵族，专门负责阅读和书写。

印刷机发明后的五十年里，欧洲出版的书籍数量超过了此前一千年的总和。书的价格跌了一百倍。

然后，花了几百年时间，识字率从 10% 涨到了全球 70%。

「我认为接下来要发生的事，」Boris 说，「是软件将被完全民主化，变成任何人都能做的事。而且会比几百年快得多。」

他举了一个很具体的例子：未来写会计软件的最佳人选，不是工程师，而是一个真正懂会计的会计——**因为懂领域才是难的，写代码变成了容易的那部分。**

这个翻转，对每一个行业都意味着什么，值得每一位企业家认真想一想。

## SaaS 的护城河正在消失——但另一批护城河更稳了

Boris 被问到一个尖锐的问题：软件开发变得这么便宜，SaaS 产品的价值还剩多少？会有「SaaS 大崩溃」吗？

他给出了一个很有框架感的回答。

他引用了投资圈著名的「七种竞争壁垒」理论（来自 Hamilton Helmer 的书），然后说：AI 会让其中一些壁垒**变弱**，但另一些壁垒**不变甚至更强**。

**变弱的壁垒：**
- **切换成本**：AI 能帮你把数据从一个系统迁移到另一个
- **流程壁垒**：过去靠复杂的内部流程形成的护城河，AI 可以直接拆解和复现

**依然稳固的壁垒：**
- **网络效应**
- **规模效应**
- **独特资源**

然后他说了让我印象很深的第二个判断：

> 未来十年诞生的颠覆性创业公司数量，将是过去十年的十倍。

因为现在，一个小团队可以用 AI 构建出大公司的产品能力，然后和那些大公司正面竞争。而大公司需要改造流程、培训员工、克服内部阻力——你们（在场的创业者）没有这个问题。

「现在是最好的创业时代，没有之一。」

## 技术差距不重要，组织差距才重要

最后一个让我反复思考的洞察，来自一个观众的问题：Anthropic 内部用的模型，和外面的人用的一样吗？

Boris 的回答出人意料：

「模型层面，我们和大家用的完全一样。我们不存在技术优势。」

「但组织层面，我们的差距可能比大多数公司大得多。」

他描述了 Anthropic 内部的日常：
- 所有人都用 Claude 写代码，包括产品经理、设计师、数据分析师、财务、用户研究员
- 代码库里没有一行人工手写的 SQL
- 不同人的 Claude Agent 之间通过 Slack 互相沟通，协调未知事项
- 所有软件，全部由模型构建

「我们领先的地方，不是技术，而是**工作方式的演化**。而这，是每一家公司都可以学的。」

这个判断，直接指向我们在做 MindsLeap 时最核心的一个相信：AI 转型，不是买一套工具，而是改变一套做事方式。

## 写在最后

Boris Cherny 是一个不太爱讲大道理的人。他在台上说的每一件事，都是自己真实在做的事。

150 个 PR 一天，几百个并发 Agent，手机上管理所有代码工作——他不是在描述未来，他在描述他的今天。

而那个「印刷机时刻」的类比，让我觉得这件事的历史重量被放到了合适的位置：

识字率从 10% 到 70%，花了几百年。

编程的民主化，可能只需要几年。

能用 AI 把想法变成软件的人，和不能的人，接下来分化的速度，会比我们想象的快很多。

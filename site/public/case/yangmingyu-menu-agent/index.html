杨铭宇菜单屏 Agent：把全国门店菜单制作变成可复制的 AI 工作流

面对连锁餐饮门店菜单屏制作高度依赖人工、设计标准难以统一、门店组合频繁变化的问题，MindsLeap 将菜单母版、菜品图库、品牌规范、版式规则和质检逻辑整合成一个可交互 Demo，为后续批量生成与门店自动化运营打样。

Business Pain

菜单屏不是一张图，而是全国门店每天都在变化的运营界面。

对连锁餐饮来说，菜单屏需要同时响应总部品牌规范、门店 SKU、价格、促销、库存和季节新品。传统方式依赖设计师逐屏制作，成本高、速度慢，也很难形成稳定的可复用流程。

1

人工制作成本高

全国门店数量越多，菜单屏尺寸、菜品组合、价格变动和活动更新越频繁，设计团队越容易被重复劳动占满。

2

品牌标准难统一

Logo、安全区、字体、描边、菜品图比例和栏目空间如果没有规则化，很容易出现不同门店视觉质量参差不齐。

3

运营反馈难闭环

菜单屏本质上是销售转化界面，但传统制作流程很难把门店销售、库存、时段和 SKU 优先级反馈进下一版画面。

Agent System

我们把“设计一张菜单屏”拆成一个 AI 原生生产系统。

Demo 阶段先验证最关键的闭环：总部母版固定、右侧内容区可重组、菜品图标准化、文本信息自动排版、生成后自动质检，并能导出可复用 PNG。

A

品牌资产 Agent

管理 Logo、色板、HarmonyOS Sans 字体、描边规则和禁用项，保证每次生成都不偏离总部视觉规范。

B

菜品图库 Agent

将黄焖、小炒、新品、配菜素材标准化，控制透明底、比例、清晰度和可投屏状态，避免门店临时找图。

C

菜单排版 Agent

根据区块容量和优先级自动组合菜品：黄焖固定 4 个，小炒 1-3 个，新品可为空，配菜 3-6 个。

D

质检 Eval Agent

检查价格缺失、图库匹配、菜品数量、图片拉伸、文字截断、品牌区域遮挡和版面拥挤度。

E

门店配置 Agent

未来接入门店 SKU、区域价格、库存、时段和活动信息，自动输出每家店可用的菜单屏版本。

F

导出与分发 Agent

将最终画面导出为标准 PNG，并为后续批量生成、审批、门店下载和投屏系统对接预留接口。

Production Workflow

从菜单数据到投屏图片，形成一条可以规模化的生产线。

这个 Demo 不是单页美术稿，而是一套“输入、匹配、排版、质检、导出”的工作流雏形，未来可以连接 Excel、门店后台或 POS 数据。

1. 输入菜单数据区块、菜名、价格、标签、优先级统一为结构化格式。
2. 匹配标准图库按菜名和类别查找真实菜品图，缺失项进入素材补齐流程。
3. 套用母版区块左侧主视觉固定，右侧按黄焖、小炒、新品、配菜区域填充。
4. 自动质检检查数量、价格、图片、品牌规范和版面拥挤度。
5. 导出投屏 PNG生成 `4000 x 2251` 横屏图片，供门店和提案场景使用。

Prototype Output

首版 Demo 交付了一个可运行、可展示、可继续工程化的样机。

现阶段重点不是一次性解决所有门店场景，而是把高频菜单屏生产问题拆成可复用组件，让客户能看见 AI Agent 进入真实运营流程的方式。

当前导出画面：左侧品牌主视觉固定，右侧自动组合黄焖系列、小炒系列、当季新品和配菜；菜名、价格、图片和区块容量由规则控制。

Eval System

真正可复制的价值，不是生成，而是每次生成都能被评估和改进。

餐饮门店菜单屏的 Eval 不只看“好不好看”，还要看是否符合品牌规范、是否可读、是否完整、是否匹配门店经营目标。

01

版式完整性

检查菜品数量是否符合区块容量，菜名和价格是否单行完整显示，图片是否拉伸或裁切。

02

品牌一致性

检查母版是否被覆盖，字体、描边、颜色、Logo 安全区和栏目标题是否保持总部标准。

03

运营有效性

未来可叠加门店销量、库存、时段和活动转化数据，把菜单屏从静态物料变成可优化的销售界面。

What Comes Next

从菜单屏 Agent，扩展为连锁餐饮门店内容生产底座。

杨铭宇菜单屏 Demo 可以继续向三个方向演进：批量门店生成、门店运营数据联动、多渠道物料自动适配。

  • 批量生成：接入 Excel / CSV 或门店后台，一次生成不同城市、门店、价格和 SKU 组合。
  • 素材治理：建立菜品图命名、抠图、清晰度、比例和版本管理规则。
  • 运营联动：将销量、库存、时段、天气和活动信息反馈到菜品排序和曝光策略。
  • 多渠道适配：从菜单屏扩展到外卖头图、团购页、门店海报、短视频封面和小红书素材。
0-2 周沉淀品牌规范、菜品图库和菜单屏母版规则。
3-6 周接入真实门店菜单数据，跑通 10-20 家门店批量生成。
7-10 周建立 Eval 看板，记录生成质量、人工修改原因和门店反馈。
11-12 周形成行业案例包，作为餐饮连锁 AI 原生运营标杆对外展示。

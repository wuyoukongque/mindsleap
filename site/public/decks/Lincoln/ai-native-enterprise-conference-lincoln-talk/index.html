  • 次日可测试上线：24 小时 / 3 个人 / 1 次 Review
定义 · 两种企业

如何定义传统企业和 AI 原生企业？不是有没有 AI 工具，而是组织能力的载体不同。

传统企业

能力在人身上
  • 经验在员工脑子里，离职即流失
  • 流程靠会议、审批、口头对齐推动
  • AI 是个人效率工具，没有进入组织系统
  • 新员工靠培训和师徒制慢慢追赶
  • 增长方式：加人、开会、层层管理

AI 原生企业

能力在系统里
  • Skill 把员工能力变成可调用资产
  • Memory 把公司数据和信息变成组织记忆
  • Workflow 按 AI 能力重新设计，而不是让 AI 填旧流程
  • Agent 持续增强，组织自动吃到执行红利
  • 增长方式：沉淀资产、复用智能、缩短判断周期

Paradigm Shift · 这不是工具升级

从 AI in the loop，
到 Human in the loop，
再到 Human on the loop。

第一阶段，是人在流程里调用 AI；第二阶段，是 AI 推进任务，人进入关键节点判断和验收。 再往后，人不再盯着每一步执行，而是设计 Spec、Harness、Gate 和 Handoff，让系统持续做对。

Anthropic Playbook · 生命周期被重启

AI 原生组织，不是更快做旧流程而是把“想法、验证、上线、规模化”的每一阶段重新设计。

01

想法

Evidence before build

People · Generalist

AI 提供高并发的执行能力。
稀缺的是能串起来的Generalist。

当专业能力可以随时调用，人的价值不再只是“会一门手艺”，而是能把业务、产品、技术、风险、客户和时机放在同一个上下文里判断。 未来的组织，需要更多能够调度 AI、整合信息、做最终判断的人。

比努力工作重要得多的时代。

Agent · 可调度的智能执行者

Agent 是执行层：模型能力要进入真实流程真正的问题不是模型强不强，而是组织能否调度、约束和验收 Agent。

  • 事实
    模型是公共服务，Agent 才是组织接口
    底层模型持续增强，但企业真正要管理的是 Agent 如何拿上下文、执行任务、交付证据。
  • 含义
    Agent 需要权限、边界和验收机制
    没有 Spec、Harness、Gate 和 Handoff，Agent 越强，组织风险也可能越大。
  • 机会
    Agent 升级自动放大组织资产
    Skill、Memory、Workflow 搭好后，Agent 能力一升级，组织整体执行能力自然增强。
  • 动作
    从真实任务里训练 Agent
    不要停留在聊天窗口，把 Agent 放进代码、文档、网页、CRM、内容流和运营流里。
Skill · 把员工技能变成 Skill

Skill 库 = 员工带不走的能力包把老员工的经验封装进系统，新人才能快速抵达可用水平。

  • 什么
    岗位 SOP 不是文档，而是可调用能力
    开场白、异议处理、合同 review、客诉回复，都可以变成 AI 一调就能执行的 Skill。
  • 为什么
    员工带不走的，才是公司的资产
    区域销售老兵 20 年经验 → 12 个 Skill → 新人 3 个月达 80% 水平。
  • 从哪
    从最贵、最容易断档的岗位开始
    公司里“如果走一个人就麻烦”的那个岗位，最值得优先 Skill 化。
  • 节奏
    每周新增 N 个 Skill
    让 Skill 像资产负债表一样被高管看见、被组织追踪、被员工贡献。
Memory · 把公司数据和信息变成 Memory

Memory 库 = 组织的长期记忆AI 看不到上下文，就只能做一次性助手。

  • 客户
    客户 Memory
    不是 CRM 字段，而是偏好、痛点、历史决策、对话片段，AI 随时调用。
  • 决策
    决策 Memory
    三年后回头看，AI 能讲清楚当时为什么选 A 不选 B。
  • 项目
    项目 Memory
    把 Spec、CLAUDE.md、决策日志和复盘教训沉淀下来，下一次 Agent 不必重新理解项目。
  • 关键
    组织透明度 = AI 员工的天花板
    公司越碎片化、越不透明，Memory 越难建；AI 看得到的，不会超过人能看到的。
IO Rate · 人与机器的速度差

机器的 IO 是并行高速。
人的 IO 仍然是会议、审批、排队。

AI 把“想到 → 做出来”的时间从几周压到几小时。新的瓶颈不在产能，而在人类审核、决策延迟和上下文切换。 AI 原生组织首先要重新审视组织结构、人员配比和 Workflow Review。

Workflow · 基于 AI 的能力重新设计

不要把 AI 塞进旧流程，要按 AI 能力重写流程流程设计的单位，从“人做什么”变成“机器先跑到哪里，人在哪里判断”。

  • 旧
    Prompt 驱动 · 人每次重写提示词
    每个任务从零开始，不可复用、不可审计、不可累积。
  • 新
    Spec 驱动 · 执行前定义可复用规则
    Goal、输入、输出、边界、验收标准先写清楚，AI 按规范执行。
  • 验收
    Harness · 让 AI 输出先被机器验收
    测试、引用检查、事实核验、业务规则先跑一轮，人只处理例外。
  • 复盘
    每周 Workflow Review
    固定问：哪些事 AI 没做？哪些人工 IO 可以砍？哪些 Spec / Harness 要沉淀？
Future Org · 更扁平，更开放，更多超级个体

未来组织会更扁平，
也会更依赖企业内部的超级个体。

金一粟在 MindsLeap 访谈里谈到硅谷精神：开放、尊重个体，相信每个人都可能成为贡献者。 AI 时代，企业内部也需要这样的环境：每个人都像合伙人一样调用智能、承担判断、创造结果。

变成整夜完成研究任务。
